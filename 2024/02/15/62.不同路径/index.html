动态规划之不同路径

发表于2024-02-15|更新于2024-08-09|算法动态规划
|字数总计:738|阅读时长:2分钟|阅读量:

力扣题目链接

题目描述

一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。
机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。
问总共有多少条不同的路径？

示例

示例1

输入：m = 3, n = 7
输出：28

示例2

输入：m = 3, n = 2
输出：3
解释：
从左上角开始，总共有 3 条路径可以到达右下角。

  1. 向右 -> 向下 -> 向下
  2. 向下 -> 向下 -> 向右
  3. 向下 -> 向右 -> 向下

示例3

输入：m = 3, n = 3
输出：6

思路

这道题很容易想到使用动态规划来求解，由于机器人要么向右走，要么向下走，所以到达位置（i,j）的路径数由到达位置(i-1,j)和(i,j-1)的路径数来决定。
使用动态规划解题步骤

确定状态数组dp

本题中因为涉及到两个坐标，所以用一个二维数组来定义状态数组dp[][]，其中dp[i][j]表示到达位置(i,j)的路径数，在dp数组中，知道里面的值具体代表什么非常重要，通常保存的是我们本题所要求的内容。

确定递归方程

在刚才其实已经把它的递归方程推导出来了，即dp[i][j] = dp[i-1][j]+dp[i][j-1];

dp数组初始状态

因为本题中涉及到两个方向的坐标，并且题目中限制了行进的方向只能往右或者往下，所以对于第一行或者第一列，其实只有一种路径。初始化dp[i][0] = dp[0][j] = 1;

确定遍历顺序

题目中已经给出了往右或者往下，那么i和j递增方向遍历

举例推导dp数组
